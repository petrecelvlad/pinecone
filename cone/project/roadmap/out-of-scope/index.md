# Out of Scope

Durable record of things considered and rejected for this project — one file per rejected *concept*, not per time it was asked. Check here before proposing something that sounds familiar; if it matches an existing entry, add an "Asked Again" line instead of re-litigating the decision or creating a duplicate file.

This is project-level (features, approaches, integrations). Framework-level rejections (declined changes to cone-lite itself) live as `REJECTED` status within individual files in [cone/evolution/](../../../evolution/index.md) instead — that's a different registry for a different pillar.
